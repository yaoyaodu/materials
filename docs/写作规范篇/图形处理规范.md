## 图形要求

技术文档中插入图形与图片建议遵循以下规范。

- 必须使用清晰可辨的图形图片。
- 中文文档里图形图片上的文字建议都用中文，如果原图文字是其他语言，应该先做好图片本地化工作。
- 文字建议使用**免费开源可商用字体**，以免引入法律风险。如不确定使用的字体是否能商用，可以在 [360 查字体](http://fonts.safe.360.cn/)网站中查证。
- 同一文档内图形图片上的中文应该统一字体（如“**思源黑体**”），英文和数字也要统一字体（如“Times New Roman”）。
- 图形图片中避免出现大段文字，描述性语言建议放到图外，用编号替代。
- 图说明和图尽量保证在同一页中显示。
- 图片的命名建议使用描述性的文字。
- **插入图片时建议添加替代文本**，从而改进文档的无障碍访问。例如，HTML 语法可以直接[添加 `alt` 属性](https://www.w3school.com.cn/html/html_images.asp)，Markdown 里使用[添加替代文本](https://markdown.com.cn/basic-syntax/images.html)，Microsoft Word 里使用[“编辑替换文字”功能](https://support.microsoft.com/zh-cn/office/向形状、图片、图表、smartart-图形或其它对象添加替换文字-44989b2a-903c-4d9a-b742-6a75b451c669)。

* 流程图不使用底色。

* 图形中包含缩略语时，需要在图下的正文说明中对缩略语进行解释。

* 所要描述的网元或网络设备，应在图形的中心位置，或在图形中突出表示。

* 图形层次结构为下级网络设备在左边或下方，上级网络设备在右边或上方的结构。

* 图应紧跟在相应的文字之后，如“如下图所示”文字之后。

* **图中描述的尺寸以毫米作为默认单位。**

* 图形中的文字应略小于正文文字。

* 绘制图的箭头风格要一致，在同一幅图中不要出现多种箭头风格，否则需要解释不同箭头的含义。

* 截图尽可能不要出现第三方软件，例如IE、PDF、Adobe Acrobat、IM等。
* 对于界面截图，禁止截图中出现鼠标，避免界面的色差与色差不均匀（通常为远程接入截图导致），禁止不突出重点地对窗口进行全部截图。

* 对于界面截图，如果是Windows系统下的截图，尽量**截取“Windows经典”主题下的操作界面**。（在桌面中点击鼠标右击，选择“个性化”，进入“主题”设置界面，双击选择“Windows经典”主题。）

## 图注

图注主要包括三种内容：

* 类型一：对图中某**部件、组成的名称**的说明。说明信息仅仅是部件、组成的名称（不做解释）。**部件、组成的位置**在图中进行了1/2/3的批注的情况。

  写作方案：使用bnl标签进行说明。禁止在bnl标签内使用句子进行说明。

  举例：下图图注的内容为图中某部件、组成的名称（不做解释），图注信息在bnl标签中说明。

  <img src="https://i.loli.net/2021/10/07/nk7CN2mXba3dxSA.png" alt="q" style="zoom: 80%;" />

* 类型二：**对于图中信息进行补充说明**。例如，图中的测量数据是在哪种环境下测得，图中某些网元未画出等信息。

  写作方案：使用Note进行说明。

  举例：下图中对于图注信息，在fig_下的note标签中说明。

  <img src="https://i.loli.net/2021/10/07/tF75m2ouwiSvCVK.jpg" alt="img" style="zoom: 67%;" />

* 类型三：对于图中一些箭头、虚线进行解释。

  写作方案：箭头或虚实线的说明作为图的一部分，使用图例进行说明。

  举例：下面的组网图中的虚线和实线的说明信息在图（image）中进行描述，作为图的一部分。

## 截图命名

界面截图的命名**需要关注截图的目的**，以下为四种典型的截图场景和对应的命名规范：

* 目的为了**展示动作进入的界面**。

  命名为： XX窗口、XX对话框、XX页面和XX对话框、XX区域框、XX设置区等。

* 目的为了**展示配置数据的示例**。

  命名为：**（动宾短语）示例**。

  例如：“配置告警条件示例”、“创建定时任务示例”。

  例如原文为“在**设置帐号**对话框中，配置DB使用的账户信息，如图1-1所示。”这里的图1-1命名为“配置DB帐户示例”。

* 目的是**需要用户完全按照截图中的数据进行配置**。

  命名为：（动宾短语）。

  例如原文为“在**设置门限**对话框中，按照如图1-2所示配置相关数据。”这里的图1-2命名为“设置门限”。

* 目的为了**展示查询某数据的查询结果**。

  命名为：查询XX。

  例如：“查询告警信息”、“查询指示灯状态”。

## 组网图

### 绘制要求

* 文本和线条的基本要求

| 类型 | **要求**                                                     |
| ---- | ------------------------------------------------------------ |
| 文本 | 中文宋体，英文和数字Arial，大小9pt，颜色黑色，字体加粗。  <br>特殊情况下，当字体大小需要进行区分时，建议为8pt~10pt。 |
| 线条 | 粗细1pt，颜色优先采用黑色。      <br/>虚线使用线形“02” （通过线条属性中“图案”项进行设置）。    <br/>点划线使用线形“04”（通过线条属性中“图案”项进行设置）。  <br/>带箭头的线端点为“13”号箭头，端点大小为“小”（通过线条属性中“线条端点”项进行设置）。 |

* 主要使用**“组网图”模块库**中的**相关图标代表网元和组网区域**；组网图中的连接线主要为**无箭头连接线**，使用“03.组网图-基本”模块库中的“14.动态连接线”连接网元。

  <img src="https://i.loli.net/2021/10/07/JSkpr8UQgimMnDY.png" alt="image-20211007093610410" style="zoom: 80%;" />

* 对于组网图中**不同线条**等图形的解释，使用“01.基本形状”模块库中的“17.**图例**”模块进行说明。

  说明：这里的**图例模块仅用于解释图中各类线条等图形的含义**，**图中其他内容的说明文字应使用图注（bnl）**，不建议使用该图例模块。图注的相关规范参见“**用户文档写作规范”→“表达方式”→“图形”**。

![image-20210823152927043](https://i.loli.net/2021/08/23/2hZj19rs5PuBQyd.png)



* 图形中避免出现大段文字，文字主要为网元名、设备名或互联方式的名称（例如Switch、TCP/IP等）即可。

* 图形中包含缩略语时，需要在正文/词汇表中对缩略语进行解释。

* 所要描述的网元或设备，应**在图形的中心位置**，或**在图形中使用橙色系图标突出表示**。

* 图形层次结构为：**下级网络设备在左侧或下方**，**上级网络设备在右侧或上方**。

* **网元名、设备名**优先放在网元图标的**正下方或正上方**；

  **互联方式的名称**优先放在连接线的**正上方或正下方**；

  **区域名称**优先放在区域框内的**左上方或右上方**。

* 文字、图标和线不要重叠。

## 界面标注图 

### 绘制要求

- 文本、线条和线框的基本要求：

| **类型**   | **要求**                                                     |
| ---------- | ------------------------------------------------------------ |
| 文本       | 英文和数字Arial，大小9pt，字体加粗。  优先采用圆形实底的数字编号。 |
| 线条和线框 | 粗细1.5pt。  颜色优先采用红色；当界面底色和红色冲突时，采用蓝色。 |

- 标注区域编号：使用**图1 标注编号**中的编号，优先选择**数字实底模块**。

## 逻辑原理图

### 绘制要求

* 文本和线条的基本要求

  | 类型 | **要求**                                                     |
  | ---- | ------------------------------------------------------------ |
  | 文本 | 中文宋体，英文和数字Arial，大小9pt，颜色黑色，字体加粗。 <br>特殊情况下，当字体大小需要进行区分时，建议为8pt~10pt。 |
  | 线条 | 粗细1pt，颜色优先采用黑色。<br>虚线使用线形“02” （通过线条属性中“图案”项进行设置）。<br>点划线使用线形“04”（通过线条属性中“图案”项进行设置）。<br>带箭头的线端点为“13”号箭头，端点大小为“小”（通过线条属性中“线条端点”项进行设置）。 |

* 主要采用“01.基本形状”模块库中的“7.**圆角矩形框**”表示**逻辑模块**。

  * 实线框：主要用于**表示小模块和重点强调的大模块**（大模块是指里面包含小模块的实体模块）。
  * 虚线框：主要用于**表示范围或非重点的大模块**。

  可以**根据需要对逻辑模块填充颜色**，颜色的配比可以参考本节中的“**颜色配比参考**”。

  ![image-20211007094858698](https://i.loli.net/2021/10/07/fDjSqpnKIJG2byv.png)

  <center><b>图 圆角矩形框</b></center>

* 主要采用“01.基本形状”模块库中的各类**箭头**标识逻辑模块间的**互联和总线信息**。

  ![image-20211007095038069](https://i.loli.net/2021/10/07/xVNUsJ3QamSPFuw.png)

* **小模块的名称**放在**模块框的正中**；**大模块或范围的名称**优先放在**模块框的左上方或右上方**。

* 图形中避免出现大段文字，主要保留模块名和互联方式的名称即可。

* 图形中包含缩略语时，需要在正文/词汇表中对缩略语进行解释。

* 框与框之间注意保持对称和对齐。

* 对于逻辑原理图中不同线条等图形的解释，使用“01.基本形状”模块库中的“17.图例”模块进行说明。
