# 钉盘架构更新

关于钉盘的初步整合建议已经完成了。为了更容易、直观的获取信息，做到信息的EASY TAKE 对部门钉盘架构做了如下调整。具体架构如“**钉盘架构图**”。

本次整合分两大块：

- 部门内部资料存档区（raw material): 保存部门团队内部的所有文档资料 、历史资料等（建议可以将现存放在SALES &Marketing 文件下的各自团队资料都转移至此文档下）。
- 部门信息规范化 整合展示区： 即团队成果内容展示，跨团队内容展示交流区域。

 

原则释义如下：

信息按照客户、内容维度进行统一整合、一体化展示。

所有存放在部门**信息规范化 整合化展示区域**的文件都是最新版本的、经过梳理和整合的。

例如：

产品文档有更新 。请把最新的产品文档放入产品文档对应区域。

**历史版本和正制作的未发布的版本**请存放在部门团队自己的文档下。

 

市场调研前期收集的资料等原始资料请存放在各自团队文档中做备份。

形成了可报告的文档内容请存放在市场前沿对应文档中。（文档会不停的矫正，请按照**只在展示区域上传最新版本的原则更新资料**。**历史版本存放在各自团队文件夹下备份**）。

烦请各位按照此原则将文档归类存放到合适的文件夹中。 有问题联系我。

部门钉盘新的文件路径：公司钉盘—Business folder。
